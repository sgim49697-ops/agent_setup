# manager_workers - 매니저/워커 멀티에이전트형

`manager_workers`는 오케스트레이터가 작업을 분해하고, 여러 워커가 병렬로 탐색 또는 구현한 뒤 매니저가 통합하는 구조입니다. 큰 과제에서 유리하지만 통합 실패가 핵심 리스크입니다.

## 기대 특성

- 강점: 탐색 병렬화, 역할 분리, 대형 과제 대응력
- 약점: 컨텍스트 전달 손실, 통합 리스크, 운영 복잡도

## 실험 포인트

- 병렬 워커 수 대비 runtime 이득을 본다.
- integration failure와 wrong assumption이 통합 단계에서 늘어나는지 관찰한다.
