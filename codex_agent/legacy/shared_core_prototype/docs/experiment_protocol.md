# experiment_protocol.md - 공정한 비교를 위한 실행 프로토콜

## 반복 규칙

- 한 시나리오당 최소 3회 반복한다.
- 평균 점수뿐 아니라 실패 사례와 표준편차도 함께 저장한다.

## 통제 항목

- 동일 모델 프로필
- 동일 토큰/호출 예산
- 동일 허용 도구 범위
- 동일 시나리오 입력

## 결과 저장 규칙

- 각 실행은 `runs/<run_id>/`에 독립 저장한다.
- `manifest.json`, `result.json`, `scorecard.json`, `summary.md`, `artifacts/`를 남긴다.
- 실패한 실행도 반드시 보존한다.

## 추천 해석 순서

1. 목표 달성 여부와 회귀 여부를 먼저 본다.
2. 그다음 요구 충족도와 검증 품질을 본다.
3. 마지막에 비용과 지연을 비교한다.

## 초기 운영 모드

- 기본은 `simulate` 모드다.
- 실제 커맨드를 연결한 실험은 같은 시나리오를 `command` 모드에서 별도 배치로 다시 실행한다.
- `long_horizon`은 첫 버전에서 체크포인트와 resume 일관성만 검증한다.
