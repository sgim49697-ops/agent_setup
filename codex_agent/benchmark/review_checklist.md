# review_checklist.md - 수동 UI/UX 리뷰 체크리스트

- 첫 화면에서 무엇을 하는 서비스인지 5초 안에 이해된다.
- 입력 폼이 과하게 복잡하지 않고, 다음 액션이 명확하다.
- 단계 진행이 자연스럽고 사용자가 길을 잃지 않는다.
- 리서치/아웃라인/초안/리뷰/최종 결과가 서로 연결되어 보인다.
- 최종 포스트가 읽기 좋고 복사/export가 자연스럽다.
- 모바일 화면에서 주요 액션이 잘리지 않는다.
- 빈 상태와 실패 상태가 허전하지 않고 안내가 충분하다.
- 시각 스타일이 의도적이고, 기본 템플릿 느낌을 벗어난다.

## UX Benchmark v2 추가 체크

- evidence 정보가 기본 사용자 화면을 압도하지 않는다.
- 사용자는 “지금 어디 단계인지”를 바로 이해할 수 있다.
- 각 단계의 primary action이 명확하다.
- 모든 산출물이 동시에 한 화면에 다 나와 있지 않아도, 전체 흐름은 이해 가능하다.
- 이 하네스만의 interaction model이 다른 하네스와 구분된다.
- 기본 사용자 가시 텍스트가 영어보다 한국어 중심으로 정리되어 있다.
